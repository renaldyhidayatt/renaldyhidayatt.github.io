## 🧩 Project Overview

**Movie gRPC** is a lightweight **gRPC microservice** built using **Rust**, with **Tonic** for the gRPC layer and **Axum** as the HTTP gateway. The project demonstrates observability practices using **OpenTelemetry**, **Jaeger**, **Prometheus**, and **Node Exporter**, all running inside a **Dockerized** environment.

All movie data is stored **in-memory**, making it perfect for demos, testing observability setups, or learning about tracing and monitoring in microservices.

---

## 🔧 Key Features

- 🧠 **In-Memory Movie CRUD**  
  Manage movie records without any external database — data resets on restart.

- 🚀 **gRPC API with Tonic**  
  Built with `tonic`, a high-performance gRPC implementation in Rust.

- 🌐 **Axum HTTP Gateway**  
  REST endpoints, healthchecks, or service introspection via `axum`.

- 🐾 **Distributed Tracing with OpenTelemetry + Jaeger**  
  See request spans and traces for gRPC calls with full context propagation.

- 📊 **Monitoring with Prometheus**  
  Collect runtime and custom metrics from your gRPC service.

- 🖥️ **System Metrics with Node Exporter**  
  Track CPU, memory, and system-level metrics from the host environment.

- 🐳 **Fully Containerized**  
  All components — app, Otel Collector, Jaeger, Prometheus, Node Exporter — run inside Docker.

## 🚀 Running Locally

```bash
git clone https://github.com/MamangRust/movie-grpc-tonic-axum.git
cd movie-grpc-tonic-axum
docker compose up --build
```
