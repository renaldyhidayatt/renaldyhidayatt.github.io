# PIERO Unified Database Management Utility

PIERO Database Management Utility (`piero-db-mgmt`) serves as a premium desktop-grade database administration console designed to unify storage lifecycle management across the PIERO LOKA ecosystem.

Built entirely with **Zed GPUI**, the system leverages full GPU acceleration to deliver a browserless, high-performance management interface. It enables fast execution of scheduled backups, large-scale relational data dumping, and real-time storage retention control without the overhead of traditional web-based database dashboards.

---

## Core Features & Architecture

### 1. GPU-Accelerated Administrative Dashboard (GPUI Core)

The interface is designed for ultra-low-latency rendering of complex database operations and system state visualization.

- **GPU Vector Rendering Engine**: All UI elements—including connection grids, backup timelines, and schema visualizations—are rendered directly using GPU primitives.
- **Zero-Latency Interaction Model**: Ensures smooth UI responsiveness even during large-scale data transfers and background backup operations.
- **High-Frequency State Updates**: Real-time synchronization of database health metrics and job execution status.

---

### 2. Polyglot Database Engine Adapters

Provides unified integration across multiple storage backends with consistent operational semantics.

- **SQLite3 Integration**: Atomic local file-based snapshots with safe locking mechanisms for concurrent access prevention.
- **PostgreSQL / MySQL Support**: Distributed dump orchestration over secure TCP connections with structured backup pipelines.
- **Redis & Key-Value Stores**: Non-blocking memory snapshot operations without interrupting active read/write workloads.
- **Extensible Adapter Layer**: Designed for future integration with additional relational and NoSQL systems.

---

### 3. FIFO Retention & Disk Lifecycle Engine

Prevents uncontrolled storage growth through deterministic retention policies.

- **Capacity Enforcement Layer**: Enforces strict storage quotas per database and backup category.
- **Deterministic Garbage Collection**: Automatically removes the oldest backup artifacts when storage thresholds are exceeded.
- **Chronological Retention Model**: Ensures predictable FIFO-based lifecycle management for archival data.

---

### 4. Asynchronous Background Job Scheduler

Automates database maintenance workflows while preserving UI responsiveness.

- **Cron-Based Scheduling Engine**: Supports precise periodic execution of backup and maintenance tasks (e.g., every 4 hours).
- **Non-Blocking Worker Runtime**: Executes compression, export, and archival pipelines in isolated asynchronous worker threads.
- **Job Isolation Model**: Ensures background workloads do not interfere with interactive database operations.

---

## Technology Stack

- **Programming Language**: Rust (Edition 2024)
- **UI Framework**: Zed GPUI GPU-accelerated rendering engine
- **Concurrency Runtime**: Tokio asynchronous task system
- **Storage Adapters**: PostgreSQL, MySQL, SQLite, Redis, MongoDB
- **Architecture Pattern**: Event-driven async job orchestration with GPU-native visualization
