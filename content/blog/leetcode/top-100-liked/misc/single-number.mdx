## Problem Statement

Given a non-empty array of integers `nums`, every element appears twice except for one. Find that single one.

You must implement a solution with a linear runtime complexity and use only constant extra space.

### Example 1:

**Input:**
```
nums = [2,2,1]
```
**Output:**
```
1
```

### Example 2:

**Input:**
```
nums = [4,1,2,1,2]
```
**Output:**
```
4
```

### Example 3:

**Input:**
```
nums = [1]
```
**Output:**
```
1
```

## Approach

- **Bitwise XOR:**
  - XOR of two same numbers is `0`.
  - XOR of a number with `0` is the number itself.
  - XORing all elements results in the single number since duplicates cancel out.

## Algorithm

1. Initialize `result = 0`.
2. Iterate through `nums` and XOR each element with `result`.
3. The final value of `result` is the single number.

## Implementations

### Implementation in Go

```go
package main
import "fmt"

func singleNumber(nums []int) int {
    result := 0
    for _, num := range nums {
        result ^= num
    }
    return result
}

func main() {
    nums := []int{4, 1, 2, 1, 2}
    fmt.Println(singleNumber(nums)) // Output: 4
}
```

### Implementation in Java

```java
class SingleNumber {
    public int singleNumber(int[] nums) {
        int result = 0;
        for (int num : nums) {
            result ^= num;
        }
        return result;
    }

    public static void main(String[] args) {
        SingleNumber sn = new SingleNumber();
        int[] nums = {4, 1, 2, 1, 2};
        System.out.println(sn.singleNumber(nums)); // Output: 4
    }
}
```

## Explanation

- **Using XOR Operation:** Efficiently finds the unique element in `O(n)` time.
- **No Extra Space Used:** Only a single variable is required.
- **Optimal Performance:** Ensures `O(n)` time complexity with `O(1)` space complexity.

## Time and Space Complexity

- **Time Complexity:** `O(n)`, as we iterate through the array once.
- **Space Complexity:** `O(1)`, since we use only one variable.
