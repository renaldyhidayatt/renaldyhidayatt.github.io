## Problem Statement

Given an integer array `nums` and an integer `k`, return the `k`th largest element in the array.

Note that it is the `k`th largest element in the sorted order, not the `k`th distinct element.

### Example 1:

**Input:**
```
nums = [3,2,1,5,6,4], k = 2
```
**Output:**
```
5
```

### Example 2:

**Input:**
```
nums = [3,2,3,1,2,4,5,5,6], k = 4
```
**Output:**
```
4
```

### Constraints:

- `1 <= k <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

## Approach

We can solve this problem efficiently using a **min-heap (priority queue)** approach or the **Quickselect algorithm**.

## Algorithm

### Min-Heap Approach:
1. Use a min-heap of size `k`.
2. Iterate through `nums`, pushing elements into the heap.
3. If heap size exceeds `k`, remove the smallest element.
4. The root of the heap is the `k`th largest element.

### Quickselect Approach:
1. Use a partitioning technique similar to **QuickSort**.
2. Partition the array such that elements greater than the pivot are on one side.
3. Recursively partition the correct side until the `k`th largest element is found.

## Implementations

### Implementation in Go

```go
package main

import (
    "container/heap"
    "fmt"
)

type MinHeap []int

func (h MinHeap) Len() int           { return len(h) }
func (h MinHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h MinHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

func (h *MinHeap) Push(x interface{}) {
    *h = append(*h, x.(int))
}

func (h *MinHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[0 : n-1]
    return x
}

func findKthLargest(nums []int, k int) int {
    h := &MinHeap{}
    heap.Init(h)
    for _, num := range nums {
        heap.Push(h, num)
        if h.Len() > k {
            heap.Pop(h)
        }
    }
    return (*h)[0]
}

func main() {
    nums := []int{3, 2, 1, 5, 6, 4}
    k := 2
    fmt.Println(findKthLargest(nums, k))
}
```

### Implementation in Java

```java
import java.util.PriorityQueue;

class Solution {
    public int findKthLargest(int[] nums, int k) {
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();
        for (int num : nums) {
            minHeap.offer(num);
            if (minHeap.size() > k) {
                minHeap.poll();
            }
        }
        return minHeap.peek();
    }
}
```

### Implementation in Python

```python
import heapq

class Solution:
    def findKthLargest(self, nums, k):
        min_heap = []
        
        for num in nums:
            heapq.heappush(min_heap, num)  # Push element to the min heap
            if len(min_heap) > k:
                heapq.heappop(min_heap)  # Remove the smallest element if heap exceeds size k
                
        return min_heap[0]  # The root of the heap is the k-th largest element

# Example usage:
sol = Solution()
print(sol.findKthLargest([3,2,1,5,6,4], 2))  # Output: 5

```




## Explanation

- **Min-Heap Approach:**
  - Maintains a heap of size `k`.
  - Ensures the smallest element in the heap is the `k`th largest.
  - Efficient for streaming data.

- **Quickselect Approach:**
  - Uses a partitioning method.
  - Average time complexity of `O(n)`.
  - Not ideal for dynamic inputs.

## Time and Space Complexity

- **Min-Heap Approach:**
  - **Time Complexity:** `O(n log k)`
  - **Space Complexity:** `O(k)`

- **Quickselect Approach:**
  - **Time Complexity:** `O(n)` (average case)
  - **Space Complexity:** `O(1)`
