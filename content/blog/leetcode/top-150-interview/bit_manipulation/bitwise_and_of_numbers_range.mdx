## Problem Statement

The **Bitwise AND of Numbers Range** problem requires finding the bitwise AND of all numbers in a given range `[left, right]`.

### Example:

```
Input: left = 5, right = 7
Output: 4
```

```
Input: left = 0, right = 0
Output: 0
```

```
Input: left = 1, right = 2147483647
Output: 0
```

### Constraints:

- `0 <= left <= right <= 2^31 - 1`

## Approach

- Use **bitwise shifting** to find the common prefix of `left` and `right`.
- Shift both numbers right until they become equal.
- Shift the result back to restore the original bit positions.

## Algorithm

1. Initialize `shift = 0`.
2. While `left != right`, shift both numbers right and increment `shift`.
3. Return `left << shift` to restore the original bit positions.

## Implementations

### Implementation in Go

```go
package main

import (
    "fmt"
)

func rangeBitwiseAnd(left int, right int) int {
    shift := 0
    for left < right {
        left >>= 1
        right >>= 1
        shift++
    }
    return left << shift
}

func main() {
    fmt.Println(rangeBitwiseAnd(5, 7))  // Output: 4
    fmt.Println(rangeBitwiseAnd(0, 0))  // Output: 0
    fmt.Println(rangeBitwiseAnd(1, 2147483647)) // Output: 0
}
```

### Implementation in Java

```java
class Solution {
    public int rangeBitwiseAnd(int left, int right) {
        int shift = 0;
        while (left < right) {
            left >>= 1;
            right >>= 1;
            shift++;
        }
        return left << shift;
    }
}
```

## Explanation

- **Bitwise Shift Right (`>>`)** removes the rightmost differing bits.
- **Bitwise Shift Left (`<<`)** restores the common prefix.
- The operation ensures only the common prefix of `left` and `right` remains.

## Time and Space Complexity

- **Time Complexity**: `O(log n)`, as we shift bits until `left == right`.
- **Space Complexity**: `O(1)`, as we use only a few integer variables.

## Conclusion

This approach efficiently finds the bitwise AND of a range using bitwise operations and shifting.
